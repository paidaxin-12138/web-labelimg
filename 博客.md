## 一款零安装、跨平台的目标检测标注工具

如果你做过目标检测，一定对 **LabelImg** 不陌生。这款经典的图像标注工具陪伴了无数算法工程师和数据标注员，它稳定、轻量，但也存在一些「成长的烦恼」：需要安装 Python/QT 环境、跨平台协作麻烦、多人标注时数据分散难以管理……

为了解决这些问题，我们开发了 **Web 版 LabelImg**，保留 LabelImg 的核心标注能力，并持续演进为可部署的协作平台。

---

## 项目简介

**Web LabelImg** 是面向目标检测数据集的 Web 图像边界框标注工具。

### 2.0 核心价值（当前主线）

- **可部署**：Docker Compose 一键启动，支持 PostgreSQL + Redis + MinIO
- **可协作**：WebSocket 实时同步光标与标注，主编辑者模型 + 乐观锁
- **可扩展**：JWT 认证、项目/成员管理、标注版本历史、YOLO 异步导出
- **更好用**：Konva 矩形编辑（拖拽 + 8 点 resize）、撤销/重做、暗黑主题

### 1.x 核心价值（轻量版，仍可用）

- **零安装**：单 HTML 页面 + FastAPI，打开浏览器即用
- **双格式保存**：JSON + YOLO TXT 同时生成
- **Session 锁协作**：适合内网小团队

## 版本对照

| 特性 | 1.x | 2.0 |
|------|-----|-----|
| 前端 | 原生 Canvas 单 HTML | Vue 3 + Konva |
| 存储 | 文件系统 | SQLite / PostgreSQL |
| 认证 | 本地姓名 | JWT 邮箱登录 |
| 协作 | 文件锁 + 15s 轮询 | WebSocket 实时广播 |
| 图像上传 | 手动放文件夹 | Web 拖拽上传 |
| 撤销/重做 | ❌ | ✅ 50 步 |
| 框编辑 | 仅新建/删 | 拖拽 + resize |
| 部署 | `python main.py` | `./start.sh` 或 Docker |

## 技术架构

### 2.0 架构

```
浏览器 (Vue 3 + Konva)
    ↕ REST / WebSocket
FastAPI (JWT + 分层服务)
    ↕
PostgreSQL / SQLite  +  Redis  +  对象存储 (local/S3)
```

### 1.x 架构

```
浏览器 (HTML5 Canvas + 原生 JS)
    ↕ REST
FastAPI 单体
    ↕
本地文件夹 (图像 + JSON + YOLO)
```

## 与 LabelImg 对比

| 特性 | LabelImg（桌面） | Web LabelImg 2.0 |
|------|------------------|------------------|
| 安装 | Python + QT | Docker 或 `./start.sh` |
| 多人协作 | 手动拷贝 | 实时 WebSocket + 项目权限 |
| 标注格式 | XML / TXT | JSON 存储 + YOLO 导出 |
| 版本管理 | 无 | annotation_versions 表 |
| 审核流程 | 无 | 规划中（Phase 3） |
